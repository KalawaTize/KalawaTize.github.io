Projet 03 - Sécurité des réseaux

Installation et configuration de pfSense dans un réseau d’entreprise

Mise en place d’un pare-feu pfSense pour contrôler les flux réseau, appliquer des règles de filtrage et améliorer la sécurité périmétrique.

Objectif du projet

Installer et configurer pfSense dans une architecture de type entreprise afin de sécuriser les communications réseau, contrôler le trafic et appliquer des règles de filtrage adaptées.

Architecture utilisée

  • Une interface WAN simulant l’accès externe.
  • Une interface LAN pour les postes internes.
  • Un pare-feu pfSense entre les deux zones.
  • Des machines clientes pour tester la connectivité.
  • Un environnement virtualisé de laboratoire.

Outils utilisés

  • pfSense
  • Virtualisation
  • Navigateur web
  • Outils de test réseau

Étapes réalisées

  1. Installation de pfSense dans l’environnement de laboratoire.
  2. Configuration des interfaces WAN et LAN.
  3. Définition des règles de pare-feu.
  4. Test de connectivité entre les zones réseau.
  5. Restriction du trafic non autorisé.
  6. Validation des règles appliquées.
  7. Documentation de la configuration.

Captures d’écran anonymisées

Résultats obtenus

  • pfSense installé et fonctionnel.
  • Segmentation de base entre les zones réseau.
  • Contrôle du trafic entrant et sortant.
  • Meilleure compréhension du rôle d’un pare-feu en entreprise.

Recommandations

  • Appliquer le principe du moindre accès.
  • Bloquer par défaut et autoriser uniquement le nécessaire.
  • Documenter toutes les règles de pare-feu.
  • Activer la journalisation des événements importants.
  • Réviser régulièrement les règles de filtrage.

Compétences démontrées

  • Sécurité réseau.
  • Configuration de pare-feu.
  • Segmentation réseau.
  • Analyse des flux.
  • Documentation d’infrastructure.
