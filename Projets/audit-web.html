Projet 04 - Test de pénétration

Audit de sécurité Web avec outils de reconnaissance et d’analyse

Audit réalisé dans un cadre autorisé afin d’évaluer l’exposition d’une application web, identifier les technologies utilisées et documenter les vulnérabilités potentielles.

Objectif du projet

Réaliser un audit de sécurité web en appliquant une méthodologie structurée : reconnaissance, énumération, analyse de surface d’attaque, test via proxy et rédaction d’observations de sécurité.

Architecture utilisée

  • Poste Kali Linux pour les tests.
  • Application ou cible web autorisée.
  • Proxy d’analyse pour observer les requêtes et réponses HTTP.
  • Réseau de laboratoire ou périmètre explicitement autorisé.

Outils utilisés

  • Recon-ng
  • theHarvester
  • Shodan
  • Proxychains et Tor
  • Nmap
  • Subfinder et Dirb
  • WhatWeb
  • Burp Suite
  • DevTools
  • Nikto
  • Nuclei
  • Kali Linux

Étapes réalisées

  1. Définition du périmètre autorisé.
  2. Collecte d’informations publiques sur la cible.
  3. Identification des sous-domaines et services exposés.
  4. Analyse des technologies web utilisées.
  5. Observation du trafic HTTP avec un proxy.
  6. Recherche de mauvaises configurations potentielles.
  7. Consolidation des résultats dans un rapport synthétique.
Les tests de sécurité web doivent toujours être réalisés sur des systèmes autorisés. Les informations sensibles, domaines et adresses IP doivent être anonymisés avant publication.

Captures d’écran anonymisées

Résultats obtenus

  • Identification de la surface d’exposition web.
  • Meilleure compréhension des outils d’audit web.
  • Détection de configurations faibles ou points d’attention.
  • Production d’une documentation structurée.

Recommandations

  • Réduire l’exposition publique inutile.
  • Maintenir les technologies web à jour.
  • Configurer correctement les en-têtes de sécurité HTTP.
  • Limiter les informations techniques divulguées.
  • Réaliser des audits réguliers dans un cadre autorisé.

Compétences démontrées

  • Reconnaissance web.
  • Énumération et cartographie.
  • Utilisation de Burp Suite.
  • Analyse de surface d’attaque.
  • Documentation d’audit.
